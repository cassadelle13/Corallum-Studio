# Terminology & Naming Map

Use these exact terms when talking to Cursor AI to ensure it finds the right components immediately.

## 1. Global Layout
- **Global Sidebar**: The main leftmost navigation menu.
- **Nav Item**: Individual buttons in the Global Sidebar (e.g., "Home Nav Item").
- **App Content**: The main area to the right of the Global Sidebar.
- **View Container**: The wrapper for different views (Home, Runs, Editor).

## 2. Home Dashboard (`HomeView`)
- **Workspace Panel**: The top informational glass panel ("Admins workspace").
- **Action Buttons**: The "Create a" buttons (+ Script, + Flow, + App).
- **Content Tabs**: The "Workspace" and "Hub" toggle buttons.
- **Filter Group**: The buttons to filter by All, Scripts, Flows, or Apps.
- **Item List**: The scrollable list of projects.
- **List Item**: An individual row in the project list.

## 3. Flow Editor (`FlowCanvas`)
- **Editor Toolbar**: The top bar inside the Flow Editor.
- **Node Library**: The left overlay panel containing draggable components.
- **Flow Canvas**: The main interactive area where nodes are placed.
- **Dot Grid**: The n8n-style background pattern on the canvas.
- **Node**: An individual functional block on the canvas.
- **Edge**: The neon animated line connecting two nodes.
- **Handle**: The connection points (dots) on the sides of a node.

## 4. Node Configuration (`NodeEditor`)
- **Node Editor Panel**: The right-side overlay for configuring a selected node.
- **Editor Tabs**: The "Code", "Inputs", and "Settings" navigation inside the panel.
- **Monaco Wrapper**: The container for the code editor.
- **Boilerplate Button**: The "Insert Boilerplate" action.
- **Data Mapping Card**: The UI elements in the "Inputs" tab for linking nodes.

## 5. Debugging (`ExecutionLogs`)
- **Logs Panel**: The bottom expandable panel showing real-time status.
- **Log Entry**: An individual line in the execution history.
- **Status Badge**: The "Success", "Running", or "Error" indicators.
